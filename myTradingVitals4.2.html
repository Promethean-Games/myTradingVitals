Data & Formatting Rules
How CSV / JSON is interpreted
▾

CSV import works best with headers like: date, ticker, direction, size, entry, exit, stop, risk, r, pnl, setup, tags, confidence, session, notes.

Unknown columns are ignored. Numbers can be decimals; notes will be flattened to a single line.

JSON import expects an array of trade objects using the same field names. On import, obvious duplicates (same date, ticker, direction, entry & exit) are automatically removed.
